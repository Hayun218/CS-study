1. **Swift에서 옵셔널(Optional)이란 무엇이며, 언제 사용해야 하나요?**

    **옵셔널**은 값이 존재할 수도 있고, 없을 수도 있는 변수의 유형입니다. 
    옵셔널을 사용하면 값이 없는 상태를 안전하게 처리할 수 있습니다.

	1-1. 옵셔널 바인딩과 강제 언래핑의 차이점은 무엇인가요?
     **옵셔널 바인딩**은 `if let` 또는 `guard let`을 사용해 안전하게 값을 추출하고, **강제 언래핑**은 `!`를 사용해 값이 반드시 존재한다고 가정하는 방식입니다.

    1-2. 옵셔널 체이닝의 동작 원리는 무엇이며, 어떻게 사용하나요?
      **옵셔널 체이닝**은 여러 옵셔널 객체를 연결해 값이 존재할 때만 접근을 허용하는 방식입니다.

    1-3. 암시적 언래핑 옵셔널(Implicitly Unwrapped Optional)은 어떤 경우에 사용해야 하나요?
      **암시적 언래핑 옵셔널**은 항상 값이 있다고 보장할 때 사용하는 옵셔널입니다.


2. **iOS 앱의 생명주기(App Life Cycle)에 대해 설명해주세요.**

	iOS 앱의 생명주기(App Life Cycle)는 앱이 실행되면서 겪는 상태 변화와 그에 따른 동작들을 관리하는 과정을 말합니다. 앱의 상태에 따라 앱이 활성화, 비활성화, 종료되는 등의 상황을 처리할 수 있습니다.
  
    2-1. 앱의 각 상태(`Not Running`, `Inactive`, `Active`, `Background`, `Suspended`)에서 가능한 작업은 무엇인가요?
    
    - **Not Running**:
	    - 앱이 실행되지 않은 상태 또는 시스템에 의해 종료된 상태입니다.
	    - 이 상태에서는 앱이 메모리에 존재하지 않으므로, 어떠한 작업도 수행할 수 없습니다.
	- **Inactive**:
	    - 앱이 화면에 표시되지만 사용자와 상호작용할 수 없는 상태입니다. 전화 수신 등으로 인해 잠시 중단되거나, 다른 알림이 활성화된 상태일 수 있습니다.
	    - 일반적으로 짧은 시간 동안 유지되며, 상태가 **Active** 또는 **Background**로 전환되기 전에 이 상태를 거칩니다.
	    - 이 상태에서는 거의 아무 작업도 수행하지 않으며, 주로 상태 전환을 준비하는 데 사용됩니다.
	- **Active**:
	    - 앱이 화면에 표시되고, 사용자와의 상호작용이 가능한 상태입니다.
	    - 대부분의 앱 로직이 이 상태에서 실행됩니다.
	    - 애니메이션 처리, 터치 이벤트 처리 등 사용자와의 실시간 상호작용이 이루어집니다.
	- **Background**:
	    - 앱이 사용자에게 보이지 않고 백그라운드에서 실행 중인 상태입니다. 앱이 이 상태에서는 일정한 작업(예: 파일 저장, 데이터 다운로드)을 수행할 수 있습니다.
	    - iOS는 특정 작업만 허용하며, 대부분의 경우 이 상태로 전환된 후 일정 시간 내에 작업을 완료하고 **Suspended** 상태로 전환됩니다.
	- **Suspended**:
	    - 앱이 백그라운드에 있지만, 코드가 실행되지 않는 상태입니다. 메모리만 차지하고 있으며, 필요할 경우 시스템에 의해 종료될 수 있습니다.
	    - 앱이 일시 정지된 상태이므로 새로운 작업을 수행할 수 없지만, 메모리에 유지되어 빠르게 재개될 수 있습니다.

    2-2. 상태 변화에 따라 호출되는 `AppDelegate` 또는 `SceneDelegate` 메서드는 무엇인가요?
    - **`didFinishLaunchingWithOptions`**: 앱 실행 후 초기화 작업.
	- **`applicationDidBecomeActive`/`sceneDidBecomeActive`**: 앱이 활성화될 때 호출.
	- **`applicationWillResignActive`/`sceneWillResignActive`**: 앱이 비활성화될 때 호출.
	- **`applicationDidEnterBackground`/`sceneDidEnterBackground`**: 백그라운드로 전환 시 호출.
	- **`applicationWillEnterForeground`/`sceneWillEnterForeground`**: 백그라운드에서 포그라운드로 전환 시 호출.
	- **`applicationWillTerminate`**: 앱 종료 전 호출.

    2-3. 백그라운드에서 작업을 완료하기 위한 방법은 어떤 것이 있나요?
    - **`beginBackgroundTask`**: 백그라운드에서 일정 시간 작업 수행 가능.
	- **Background Fetch**: 시스템이 주기적으로 앱을 깨워 작업 처리.
	- **Silent Notification**: 푸시 알림을 통해 백그라운드에서 작업 실행.

4. **Swift에서 클로저(Closure)란 무엇이며, 어떻게 사용하나요?**

	클로저(Closure)는 Swift에서 **독립적인 코드 블록**으로, 함수와 유사하지만 더 경량화된 문법을 가진 기능입니다. 이를 코드의 캡슐화와 재사용성을 높이는 데 활용합니다.

	4-1. 클로저의 캡처(Capture) 기능은 무엇인가요?
	
	클로저는 정의된 컨텍스트의 상수와 변수를 캡처하여 저장할 수 있습니다. 이를 통해 클로저가 생성된 시점의 값들을 나중에도 참조하고 수정할 수 있습니다. 값 타입은 복사되어 캡처되고, 참조 타입은 참조로 캡처되는 특징이 있습니다.

    4-2.  @escaping 클로저와 non-escaping 클로저의 차이점은 무엇인가요?
    
	-  non-escaping: 기본값으로, 함수의 실행이 종료되기 전에 클로저의 실행 보장	
	- @escaping: 함수가 종료된 후에도 클로저가 실행될 수 있음을 나타내며, 주로 비동기 작업이나 클로저를 프로퍼티로 저장할 때 사용
	- 성능 최적화를 위해 필요한 경우에만 @escaping을 사용하는 것이 권장된다.

    4-3. 트레일링 클로저(Trailing Closure) 문법은 어떤 경우에 유용한가요?
    
	- 함수의 마지막 파라미터가 클로저일 때 사용할 수 있는 문법입니다.
	- 코드의 가독성을 높이고, 긴 클로저를 더 깔끔하게 표현할 수 있으며, 특히 비동기 작업이나 콜백 함수를 구현할 때 매우 유용합니다.
	- 가장 흔히 볼 수 있는 예시는 애니메이션:
```swift
// 일반적인 방식
UIView.animate(withDuration: 0.3, animations: {
    view.alpha = 0
})

// 트레일링 클로저 사용
UIView.animate(withDuration: 0.3) {
    view.alpha = 0
}
```


5. **iOS에서 Delegate 패턴은 무엇이며, 어떤 상황에서 사용되나요?**

    Delegate 패턴은 iOS에서 객체 간 통신을 위해 사용되는 디자인 패턴입니다. 주로 한 객체가 다른 객체를 대신하여 동작하거나, 특정 이벤트 발생 시 이를 위임받은 객체에게 알리는 용도로 사용됩니다.

    5-1. Delegate 패턴과 Notification, KVO의 차이점은 무엇인가요?
    
    **Delegate 패턴:**
	- 1:1 통신 방식으로, 명확하고 직접적인 객체 간 통신이 가능합니다.
	- 컴파일 타임에 오류를 체크할 수 있어 안정적입니다.
	- 프로토콜을 통해 필요한 메소드가 명확히 정의됩니다.
	
	**Notification:**
	- 1:N 브로드캐스팅 방식으로, 다수의 객체에게 동시에 이벤트를 전달할 수 있습니다.
	- 느슨한 결합으로 유연하지만, 디버깅이 상대적으로 어렵습니다.
	- 문자열 기반의 이벤트 전달로 타입 안정성이 떨어집니다.
	
	**KVO(Key-Value Observing):**
	- 특정 객체의 프로퍼티 변경을 관찰하는 데 특화되어 있습니다.
	- NSObject를 상속해야 하는 제약이 있습니다.
	- Objective-C 런타임에 의존적입니다.

    5-2. 프로토콜을 활용한 Delegate 패턴 구현 방법을 설명해주세요.
	
	- 프로토콜을 정의하여 필요한 메소드 선언합니다.
	- delegate 프로퍼티는 메모리 누수 방지를 위해 weak로 선언합니다.
	- 프로토콜은 AnyObject를 상속하여 클래스 전용으로 제한합니다.
	- delegate 메소드 호출 시 옵셔널 체이닝을 사용합니다.

13. **Swift의 고차 함수(Higher-Order Functions)에 대해 설명해주세요.**

    다른 함수를 파라미터로 받거나 함수를 반환하는 함수를 말합니다. 컬렉션 타입의 데이터를 효율적으로 처리할 수 있게 해줍니다.
    코드의 가독성을 높이고 함수형 프로그래밍의 장점을 살려 안전하고 효율적인 데이터 처리를 가능하게 합니다. 특히 체이닝을 통해 여러 연산을 조합할 수 있어 복잡한 데이터 처리도 간결하게 표현할 수 있습니다.

    13-1. `map`과 `flatMap`의 차이점은 무엇인가요?
    
    - map: 컬렉션의 각 요소를 변환하여 새로운 배열을 생성하며 각 요소는 1:1로 매핑
	- flatMap: 중첩된 컬렉션을 단일 수준으로 평탄화하면서 변환 
		- 주로 2차원 배열을 1차원으로 만들거나 옵셔널 체이닝에서 사용
	
    13-2. `filter`, `reduce` 함수는 어떤 경우에 사용하나요?
    
    - filter: 컬렉션에서 특정 조건을 만족하는 요소만을 선택하여 새로운 배열 생성 
	    - 주로 검색이나 데이터 필터링에 사용
	- reduce: 컬렉션의 모든 요소를 하나의 값으로 결합 
		- 합계 계산이나 문자열 연결 등 컬렉션의 요소들을 누적해서 처리할 때 사용

    13-3. `compactMap`은 어떤 역할을 하나요?
    
    - compactMap: 컬렉션의 nil 값을 제거하고 옵셔널 바인딩을 처리
		- 변환 과정에서 실패할 수 있는 경우(예: 문자열을 숫자로 변환)에 유용하며, 유효한 값만 추출할 때 사용

26. **동시성 프로그래밍에서 동기(Synchronous)와 비동기(Asynchronous)의 차이점은 무엇인가요?**

	동시성 프로그래밍(Concurrent Programming)은 여러 작업을 동시에 처리할 수 있게 해주는 프로그래밍 패러다임입니다. 동기(Synchronous)와 비동기(Asynchronous)는 작업 처리 방식의 차이를 나타냅니다.
	
	**동기 처리:** 
	- 작업이 순차적으로 실행됩니다 
	- 현재 작업이 완료될 때까지 다음 작업이 블록됩니다 
	- 코드의 실행 흐름이 예측 가능합니다 
	
	**비동기 처리:** 
	- 작업이 병렬적으로 실행될 수 있습니다 
	- 현재 작업이 완료되기를 기다리지 않고 다음 작업을 실행합니다 
	- 완료 시점에 콜백이나 completion handler를 통해 결과를 처리합니다

    26-1. iOS에서 비동기 작업을 처리하는 방법은 무엇인가요?
    1. GCD (Grand Central Dispatch) 
    2. Operation Queue 
    3. async/await (iOS 15 이상) 
    4. Completion handlers 
    5. Combine 프레임워크

    26-2. 세마포어(Semaphore)와 뮤텍스(Mutex)의 차이점은 무엇인가요?
    - 세마포어: 여러 개의 리소스를 관리할 수 있습니다. => `s = 1` 일 경우 뮤텍스와 동일
    - 뮤텍스: 단일 리소스의 접근을 관리하며, 소유권 개념이 있습니다


27. **GCD(Grand Central Dispatch)의 주요 개념과 사용 방법을 설명해주세요.**

    GCD(Grand Central Dispatch)는 iOS에서 멀티스레딩을 쉽게 구현할 수 있게 해주는 저수준 API입니다. 작업을 대기열(Queue)에 추가하고 시스템이 알아서 스레드를 관리해주는 방식으로 동작합니다.
    
	**GCD 사용 시 주의사항:**
	1. UI 업데이트는 반드시 메인 큐에서 실행
	2. 데드락 방지를 위한 적절한 큐 선택
	3. 무거운 작업은 적절한 QoS 레벨 선택
	4. 비동기 작업의 순서 관리 필요

    27-1. 직렬(Serial) 큐와 동시(Concurrent) 큐의 차이는 무엇인가요?
    
	**직렬 큐(Serial Queue):**
	- 작업을 순차적으로 하나씩 실행
	- FIFO(First-In-First-Out) 순서 보장
	- 데이터 경쟁 조건 방지에 유용
	
	**동시 큐(Concurrent Queue):**
	- 여러 작업을 동시에 실행 가능
	- 작업 완료 순서는 보장되지 않음
	- 처리량이 많은 작업에 적합

    27-2. 글로벌 큐(Global Queue)와 메인 큐(Main Queue)는 어떻게 다르나요?
    
	**글로벌 큐:**
	- 시스템에서 제공하는 동시 큐
	- QoS 우선순위 지정 가능
	- 백그라운드 작업에 주로 사용
	
	**메인 큐:**
	- UI 업데이트를 위한 직렬 큐
	- 모든 UI 관련 작업은 메인 큐에서 실행
	- 메인 스레드에서 실행됨
	

    27-3. DispatchWorkItem을 사용하는 방법은 무엇인가요?
    
    - 재사용 가능한 작업 단위
	- 작업 취소 가능
	- 작업 완료 후 콜백 설정 가능
```swift
let workItem = DispatchWorkItem {
    print("작업 실행")
}

// 작업 실행
DispatchQueue.global().async(execute: workItem)

// 작업 취소
workItem.cancel()

// 완료 통지
workItem.notify(queue: .main) {
    print("작업 완료")
}
```
